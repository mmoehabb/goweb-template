# Instructions

Follow these instructions at the start of every session.

## Session Rules

1. **Read memory first** - Always read `.agents/memory.md` at the start of each session to understand project context and decisions.

2. **Update todos** - Before starting work, review `.agents/todos.md`. Mark completed tasks and add any new tasks you identify.

3. **Write to memory** - After each session, write important decisions, context, and notes to `.agents/memory.md` for future sessions.

4. **Check instructions** - Read `.agents/instructions.md` at session start to refresh these rules.

## General Guidelines

- Explore the codebase first to understand existing patterns
- Follow conventions defined in `.agents/guide.md` and `.agents/architecture.md`
- Use existing packages and patterns when possible
- Don't introduce new dependencies without good reason
- Keep changes focused and minimal

## Workflow

1. Read memory to understand context
2. Review todos to see what needs doing
3. Read relevant guide/architecture files
4. Make changes
5. Update memory with new context
6. Update todos to reflect progress
