# Architecture

The GoWeb template is a web application built with Go Fiber, using a-h/templ for HTML rendering and pgx for PostgreSQL database connectivity.

## Directory Structure

```
goweb-template/
├── main.go              # Application entry point, router setup
├── handlers/           # HTTP request handlers
│   └── user/          # User-related handlers (login, register)
├── db/                # Database layer
│   ├── db.go         # Core db connection, RunMigrations(), Queries()
│   ├── migrations/   # SQL migration files (goose)
│   └── users/        # User-specific database operations
├── ui/                # UI components (Templ)
│   ├── components/   # UI components (templui + custom)
│   │   ├── button/   # templui button component
│   │   └── TextInput.templ
│   ├── forms/        # Form components (login, register)
│   ├── layouts/      # Page layouts (header, footer, page wrapper)
│   └── utils/        # templui utilities
│       └── templui.go
├── pages/            # Page templates (auto-discovered by endpoints.go)
├── ancillaries/      # Utilities (endpoint discovery, error handling)
├── constants/        # Configuration (config.go loads .env)
└── public/           # Static assets
```

## Package Dependencies

```
main.go
  ├── fiber (github.com/gofiber/fiber/v2)      # HTTP router
  ├── godotenv (github.com/joho/godotenv)     # .env loader
  ├── pgx pool (github.com/jackc/pgx/v5)      # PostgreSQL
  ├── ancillaries (local)                      # Utilities
  ├── constants (local)                      # Config
  ├── handlers (local)                       # Routes to handlers
  └── pages (local)                         # Render pages

handlers/user/
  ├── fiber                                   # Context & status codes
  ├── users (local)                          # Database operations
  └── forms (local)                          # Render forms

db/
  ├── database/sql                          # Standard SQL interface
  ├── pgx/stdlib (github.com/jackc/pgx/v5) # PostgreSQL driver
  └── goose (github.com/pressly/goose/v3)  # Migrations

ui/*/ (generated by templ)
  ├── templ (github.com/a-h/templ)           # HTML components
  ├── components (local)                     # Custom UI components
  └── button (local)                         # templui button from components/

components/*/ (templui)
  ├── templ (github.com/a-h/templ)
  └── utils (local)                          # templui utilities
```

## How Routes Flow

1. `main.go` sets up Fiber app
2. Routes are registered: `app.Post("/login", user.Login)`
3. Handler reads request, validates, calls db layer
4. Handler renders UI component via Templ
5. Templ writes HTML to response writer

## Data Flow

```
HTTP Request
    ↓
Handler (handlers/*/)
    ↓
Database (db/*/)
    ↓
Query PostgreSQL (pgx)
    ↓
Return data to Handler
    ↓
Render UI (ui/*/)
    ↓
HTTP Response
```
